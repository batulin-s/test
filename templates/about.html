HTTP. Basically this application will serve several endpoints to meet following requirements:

1 POST /classes should create a class
Request should be of the form:

{

"name": "Test class", // class name, unique

"code": "AbcdefgHIJ", // unique, read-only, 10-letter code

"teacher": "http://user-service/12345", // Teacher's URI

"students": [ // List of students' URIs

"http://user-service/111111",

"http://user-service/222222"

]

}
Request to create class with unique fields that have been already in use raised
409 Http error (Conflict)

Request with invalid parameters caused to 406 http error (Not Acceptable)

a This endpoint should accept query parameters to filter results. For example GET

/classes?student=XXX should return classes with XXX student enrolled.

2 GET /classes should return list of all classes

3 GET /classes/<code> should return single class

4 PUT /classes/<code> should replace existing class with provided object
{

"name": "Test class", // class name, unique

"teacher": "http://user-service/12345", // Teacher's URI

"students": [ // List of students' URIs

"http://user-service/111111",

"http://user-service/222222"

]

}
Attention! PUT request shuldn't contains code parameter!)

5 DELETE /classes/<code> should delete class
